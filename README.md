# Module Eight Journal: Portfolio Item

This repository contains my Project Two dashboard application and the accompanying README documentation created for the CS‑340 course. These artifacts demonstrate my ability to design, implement, and document a database‑driven web application using MongoDB, Python, and the Dash framework.

## Writing Maintainable, Readable, and Adaptable Programs

I write maintainable and adaptable programs by separating responsibilities and keeping code modular. This approach was especially important when developing the CRUD Python module in Project One and reusing it in Project Two. By isolating database logic inside the CRUD module, I avoided duplicating code and kept the dashboard logic clean and readable.

The primary advantage of this approach is reusability. The same CRUD module can be reused in future applications such as APIs, command‑line tools, or additional dashboards. In the future, this module could be extended to support features like authentication, logging, or more advanced queries without modifying the dashboard code.

---

## Problem-Solving Approach as a Computer Scientist

I approached this project by first understanding the client’s requirements and then breaking the problem into smaller components: database setup, query design, dashboard layout, and interactivity. This differed from earlier coursework that focused mainly on isolated programming tasks, as this project required a system‑level perspective.

Testing database queries independently before integrating them into the dashboard helped identify issues early and made troubleshooting more efficient. In future projects, I would continue using this structured approach to design databases that meet specific client needs.

## The Role of Computer Scientists and Why It Matters

Computer scientists design tools that help organizations make better use of their data. In this project, the dashboard transforms raw animal shelter data into an interactive tool that allows Grazioso Salvare to quickly identify dogs suitable for rescue training.

This type of work matters because it improves efficiency, reduces manual effort, and supports data‑driven decision‑making. For an organization like Grazioso Salvare, this means spending less time managing data and more time focusing on its mission.
